### ⚠️ 遇到的问题及解决方案
1. **Vite 默认脚手架预设问题与 TypeScript 报错**
   - **错误描述**：初始创建命令因为参数交互问题，最终输出了纯 TS 的 Vanilla 模板。导致开发 `index.tsx` 时抛出大量 `Cannot use JSX unless the '--jsx' flag is provided` 报错，且无法启动 React。
   - **解决方案**：手工移除了无用的 `main.ts` 等文件，在 `tsconfig.json` 中补齐了 `"jsx": "react-jsx"` 及其 `include` 映射；在 `package.json` 补装了 `@vitejs/plugin-react`、`@types/react` 等预设，并修正了 `index.html` 中的挂载点（`<div id="root">` 以及 `<script src="...main.tsx">`）。
2. **Tailwind CSS v4 在 Vite 中构建机制变化**
   - **错误描述**：在按旧方案引入 `tailwindcss` 到 `postcss.config.js` 后，Vite 打包时抛出：`It looks like you're trying to use tailwindcss directly as a PostCSS plugin... update your PostCSS configuration.`
   - **解决方案**：将配置迁移至 Tailwind CSS V4 推荐的方法，即安装并使用配套的 `@tailwindcss/postcss` 引擎重写 postcss 配置文件，并在 `index.css` 改用 `@import "tailwindcss";`。
3. **严格模式下未使用的变量拦截构建**
   - **错误描述**：`npm run build` 时 `tsc` 拦截到诸如 `useEffect` 和 `currentSlide` 等未使用变量声明导致打包阻断。
   - **解决方案**：及时移除了冗余的引用变量并将无关代码精简。
