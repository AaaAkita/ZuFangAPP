## 4. 数据库设计规划

**详细文档**：`doc/06-数据库设计.md`

### 4.1 核心表结构

| 表名 | 主键 | 索引 | 关联 |
|------|------|------|------|
| users | id | phone, email | 1:N comments, 1:N search_history |
| communities | {id} | name, location | 1:N comments, 1:N ads |
| comments | id | communityId, userId | N:1 users, N:1 communities |
| comment_likes | id | commentId, userId | N:1 comments, N:1 users |
| search_history | id | userId, keyword | N:1 users |
| advertisements | id | position, status | - |
| admin_users | {id} | username | - |

### 4.2 字段设计原则

1. **ID字段**：统一使用 `BigInt`，类型安全
2. **时间字段**：统一使用 `DateTime`，含时区
3. **软删除**：重要表添加 `deletedAt` 字段
4. **状态字段**：使用枚举类型，便于维护
5. **索引设计**：查询频繁字段添加索引

## 5. 认证模块设计

**详细文档**：`doc/07-认证模块.md`

### 5.1 认证流程

```
用户注册/登录
    ↓
验证用户信息
    ↓
生成JWT Token
    ↓
返回Token给客户端
    ↓
客户端存储Token
    ↓
后续请求携带Token
    ↓
服务端验证Token
    ↓
允许访问
```

### 5.2 JWT配置

```typescript
{
  secret: 'your-secret-key',    // 密钥
  expiresIn: '7d',              // 有效期
  algorithm: 'HS256'            // 算法
}
```

### 5.3 守卫设计

- **JwtAuthGuard**：验证JWT Token
- **AdminGuard**：验证管理员权限
- **@User()装饰器**：获取当前用户信息

## 12. 开发注意事项

### 12.1 后端开发

1. **类型安全**：使用TypeScript，避免any类型
2. **错误处理**：统一使用异常过滤器
3. **数据验证**：使用class-validator验证DTO
4. **日志记录**：重要操作记录日志
5. **性能优化**：查询使用索引，避免N+1问题

### 12.2 前端开发

1. **状态管理**：使用Pinia管理全局状态
2. **API封装**：统一封装请求，处理错误
3. **类型定义**：定义接口类型，避免类型错误
4. **性能优化**：列表使用虚拟滚动，图片懒加载
5. **用户体验**：加载状态、错误提示、空状态处理

### 12.3 数据库开发

1. **迁移管理**：使用Prisma管理数据库变更
2. **种子数据**：准备测试和演示数据
3. **索引优化**：查询频繁字段添加索引
4. **数据备份**：定期备份，防止数据丢失
