# 租房避雷App - 数据库设计

## 1. 数据库概述

- **数据库类型**：PostgreSQL 15+
- **ORM框架**：Prisma
- **字符集**：UTF-8
- **时区**：UTC

## 2. 数据表设计

## 3. 数据关系图

```
User (用户)
  ├── 1:N Comment (评论)
  ├── 1:N CommentLike (点赞)
  └── 1:N SearchHistory (搜索历史)

Community (小区)
  ├──   1:N Comment (评论)
  └── 1:N Advertisement (广告)

Comment (评论)
  ├── N:1 User (评论人)
  ├── N:1 Community (所属小区)
  └── 1:N CommentLike (点赞)

CommentLike (点赞)
  ├── N:1 User (User)
  └── N:1 Comment (评论)

SearchHistory (搜索历史)
  └── N:1 User (用户)

Advertisement (广告)
  └── N:1 Community (所属小区)

AdminUser (管理员)
  ├── 1:N Report (处理的举报)
  └── 1:N Contract (验证的合同)

Report (举报)
  ├── N:1 User (举报人)
  └── N:1 AdminUser (处理人)

Contract (合同)
  ├── N:1 User (用户)
  ├── N:1 Comment (关联评论)
  └── N:1 AdminUser (审核人)
```

## 4. 数据库优化建议

### 4.1 索引优化

1. **复合索引**：对于经常一起查询的字段创建复合索引
   - `(city, district, type)` 用于小区筛选
   - `(communityId, createdAt)` 用于评论列表

2. **部分索引**：对于特定条件的数据创建部分索引
   - 仅对 `isRecommended=true` 的小区创建推荐索引

### 4.2 分区策略

当数据量大时，考虑按时间分区：
- `comments` 表按 `createdAt` 分区
- `search_history` 表按 `createdAt` 分区

### 4.3 查询优化

1. **避免N+1问题**：使用 Prisma 的 `include` 或 `select` 关联查询
2. **分页优化**：使用 cursor-based 分页代替 offset-based
3. **只查必要字段**：避免 `SELECT *`

### 4.4 数据归档

- 历史搜索数据定期归档（超过6个月的数据）
- 已删除的数据保留软删除记录，定期物理清理

## 5. 数据迁移

### 5.1 迁移文件位置

```
backend/prisma/migrations/
```

### 5.2 迁移命令

```bash
# 创建迁移
npx prisma migrate dev --name migration_name

# 应用迁移
npx prisma migrate deploy

# 重置数据库（开发环境）
npx prisma migrate reset

# 查看迁移状态
npx prisma migrate status
```

### 5.3 种子数据

npx prisma db seed

## 6. 数据备份

### 6.1 备份策略

| 备份类型 | 频率 | 保留时间 |
|---------|------|---------|
| 全量备份 | 每天 | 30天 |
| 增量备份 | 每小时 | 7天 |
| Binlog | 实时 | 24小时 |

### 6.2 备份命令

```bash
# 全量备份
pg_dump -U username -d database_name > backup.sql

# 恢复备份
psql -U username -d database_name < backup.sql
```

### 6.3 容灾方案

- 主从复制：实时同步到从库
- 异地备份：每日备份到异地存储
- 恢复演练：每月演练一次数据恢复
