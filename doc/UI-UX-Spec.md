## 6. 后端API设计规范

### 6.1 RESTful API规范

#### 6.1.1 URL设计
```
基础URL：https://api.zufang.com/v1

资源命名：复数形式
- /users
- /communities
- /comments

层级关系：
- /communities/{id}/comments
- /users/{id}/contracts
```

#### 6.1.2 HTTP方法
| 方法 | 用途 | 示例 |
|------|------|------|
| GET | 获取资源 | GET /users/1 |
| POST | 创建资源 | POST /users |
| PUT | 更新资源 | PUT /users/1 |
| DELETE | 删除资源 | DELETE /users/1 |
| PATCH | 部分更新 | PATCH /users/1 |

#### 6.1.3 状态码
| 状态码 | 含义 | 使用场景 |
|--------|------|----------|
| 200 | 成功 | GET、PUT、PATCH |
| 201 | 创建成功 | POST |
| 204 | 无内容 | DELETE |
| 400 | 请求错误 | 参数验证失败 |
| 401 | 未认证 | Token过期 |
| 403 | 无权限 | 权限不足 |
| 404 | 资源不存在 | 资源未找到 |
| 500 | 服务器错误 | 服务器异常 |

### 6.2 数据格式规范

#### 6.2.1 请求格式
```json
{
  "data": {
    "name": "张三",
    "phone": "13800138000"
  }
}
```

#### 6.2.2 响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "name": "张三"
  },
  "timestamp": 1640000000000
}
```

#### 6.2.3 错误响应格式
```json
{
  "code": 400,
  "message": "参数验证失败",
  "errors": [
    {
      "field": "phone",
      "message": "手机号格式不正确"
    }
  ],
  "timestamp": 1640000000000
}
```

#### 6.2.4 分页响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "items": [],
    "pagination": {
      "page": 1,
      "pageSize": 20,
      "total": 100,
      "totalPages": 5
    }
  },
  "timestamp": 1640000000000
}
```

### 6.3 数据库字段规范

#### 6.3.1 命名规范
- 表名：小写，下划线分隔（如：user_profiles）
- 字段名：小写，下划线分隔（如：created_at）
- 主键：id（自增）
- 外键：{表名}_id（如：user_id）

#### 6.3.2 通用字段
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 主键 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |
| deleted_at | TIMESTAMP | 删除时间（软删除） |

#### 6.3.3 时间格式
- 数据库：UTC时间
- API响应：ISO 8601格式（如：2024-01-01T00:00:00Z）
- 前端展示：本地时间

### 6.4 认证授权规范

#### 6.4.1 JWT Token
```
Header: Authorization: Bearer {token}
Token有效期：7天
刷新Token：30天
```

#### 6.4.2 权限控制
| 角色 | 权限 |
|------|------|
| 普通用户 | 浏览、评论、点赞 |
| 认证用户 | 浏览、评论、点赞、发布真实评论 |
| 管理员 | 所有权限 |
