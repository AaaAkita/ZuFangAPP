## 2. 技术栈

### **2.1 前端技术栈**

| 技术 | 版本/说明 | 用途 |
|------|----------|------|
| uni-app | Vue 3 + TypeScript | 跨平台框架 |
| Vite | - | 构建工具 |
| TypeScript | ^5.x | 类型安全 |
| Pinia | - | 状态管理 |
| Uni-UI | - | UI组件库 |

**前端架构特点**：
- 一套代码多端运行
- 优先开发H5和微信小程序（P0）
- 后续扩展iOS、Android（P2）
- 响应式设计，适配不同屏幕尺寸
- 组件化开发，代码复用率高

### **2.2 后端技术栈**

| 技术 | 版本/说明 | 用途 |
|------|----------|------|
| Node.js | ^18.x | 运行时环境 |
| NestJS | ^10.x | 应用框架 |
| TypeScript | ^5.x | 类型安全 |
| Prisma | ^5.x | ORM |
| PostgreSQL | ^15.x | 主数据库 |
| Redis | ^7.x | 缓存/会话 |
| Passport | - | 认证中间件 |
| JWT | - | Token认证 |

**后端架构特点**：
- 模块化设计，易于扩展
- 依赖注入（DI），便于测试
- 统一异常处理和响应拦截
- 完善的守卫、拦截器、管道体系

## 5. 数据库设计

### 5.1 核心数据表

| 表名 | 说明 | 主要字段 |
|------|------|---------|
| users | 用户表 | id, phone, email, nickname, avatar |
| communities | 小区表 | name, address, type, rating, location |
| comments | 评论表 | content, userId, communityId, rating |
| comment_likes | 点赞表 | commentId, userId |
| search_history | 搜索历史 | userId, keyword |
| advertisements | 广告表 | title, image, link, position |
| admin_users | 管理员表 | username, password, role |

### 5.2 数据关系

```
User (用户)
  ├── 1:N Comment (评论)
  ├── 1:N CommentLike (点赞)
  └── 1:N SearchHistory (搜索历史)

Community (小区)
  ├── 1:N Comment (评论)
  └── 1:N Advertisement (广告)

Comment (评论)
  ├── N:1 User (评论人)
  ├── N:1 Community (所属小区)
  └── 1:N CommentLike (点赞)
```

## 6. API设计规范

### 6.1 路由规范

- **全局前缀**：`/api/v1`
- **RESTful风格**：使用标准HTTP方法
  - GET：查询资源
  - POST：创建资源
  - PUT/PATCH：更新资源
  - DELETE：删除资源

### 6.2 响应格式

**成功响应**：
```json
{
  "success": true,
  "data": { ... },
  "message": "操作成功",
  "timestamp": 1729837261893
}
```

**失败响应**：
```json
{
  "success": false,
  "error": {
    "statusCode": 401,
    "message": "Token 已过期",
    "path": "/api/v1/users/profile"
  },
  "timestamp": 1729837265000
}
```

### 6.3 认证方式

- **JWT Token**：有效期7天
- **Token位置**：Header `Authorization: Bearer <token>`
- **刷新机制**：Token过期需重新登录

## 8. 安全设计

### 8.1 数据安全
- 用户密码使用 `bcrypt` 加密存储
- 敏感信息（合同、身份证）脱敏处理
- HTTPS传输加密

### 8.2 接口安全
- JWT认证，Token过期自动失效
- 接口频率限制，防止恶意调用
- SQL注入防护（Prisma自动防护）

### 8.3 内容安全
- 用户上传文件大小限制
- XSS攻击防护
- CSRF攻击防护
