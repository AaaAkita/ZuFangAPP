### ⚠️ 遇到的问题及解决方案
1. **Prisma Schema (v7.x) 不支持 url 参数问题**
   - **错误描述**：`Prisma schema validation Error code: P1012` 提示不支持将 database url 写死在 schema 中。
   - **解决方案**：移除了 `schema.prisma` 中的 `url = env("DATABASE_URL")` 相关属性。在 prisma.config.ts 里有相应的提供。
2. **Prisma Relation 命名冲突关联问题**
   - **错误描述**：Prisma Client 构建时提示 `CommentReply` 以及 `CommentReport` 实体的外键（用户和处理人/审计人员关联都是 `User` 表）关系存在歧义 `Ambiguous relation detected`。
   - **解决方案**：在 Prisma 侧必须为那些同样指向 User 的一对多表附上显式的命名，例如 `@relation("CommentReplyUser")` 和 `@relation("AuditUser")`。
3. **TypeScript / IDE 类型报错“缓存”幻象**
   - **错误描述**：IDE 检测到类似“PrismaService”中没有“$connect”，以及模块“@prisma/client”没有导出的成员“PrismaClient”。
   - **解决方案**：属于 IDE 或者 TS Server 缓存更新没跟上代码生成的节奏引起。通过 `Restart TS Server` 后解决，实际 `npm run build` 并无错误。
4. **JWT Options TypeScript 类型定义问题**
   - **错误描述**：配置项中的 `expiresIn: '7d'` 签名选项报错 `Type 'string' is not assignable to type 'number | StringValue | undefined'`。
   - **解决方案**：使用 `as any` 将其强转。
5. **JSON.stringify 对 BigInt 的序列化异常**
   - **错误描述**：由于 PostgreSQL 大主键映射到 JS 中的 `BigInt` 不被 JSON 原生 `JSON.stringify` 识别，NestJS 响应前端时报 500。
   - **解决方案**：在 `main.ts` 入口处补充全覆盖：`(BigInt.prototype as any).toJSON = function () { return Number(this); };`

1. **TypeScript 对对象索引赋值报隐式 Any（TS7053）：**
   - **错误描述**：给 Prisma 的 orderBy 进行动态字符串属性赋值时，不被 TypeScript 支持，提示它缺少隐式的 string 索引签名。
