# 租房避雷App - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景
开发一款基于云服务器的租房避雷App，支持手机端和网页端，帮助用户了解真实房源信息，避免租房踩坑。

### 1.2 项目目标
- 提供真实可靠的房源评价平台
- 通过合同验证机制防止虚假评论
- 建立租房信息共享社区
- 通过广告位实现商业变现

### 1.3 目标用户
- 正在租房的租客
- 有租房经验的租客
- 房东
- 房产中介

---

## 2. 用户体系

### 2.1 用户注册登录
- 手机号注册登录
- 邮箱注册登录
- 第三方登录：微信

### 2.2 用户角色
- **普通用户**：浏览、评论、点赞
- **认证用户**：通过合同验证，评论标记为真实评论
  - 认证类型：租客、房东、房产中介
- **管理员**：内容审核、用户管理

### 2.3 个人中心功能
- 查看历史评论
- 查看合同验证状态
  - 验证状态：已验证/未验证
  - 核验合同日期与失效日期
  - 接入OCR识别
- 个人信息管理
- 消息通知

---

## 3. 核心功能模块

### 3.1 房源定位功能
#### 3.1.1 小区选择
- 搜索小区名称
- 地图定位选择
- 热门小区推荐
- 地图服务：高德地图API
- 服务范围：北京、天津、河北三市


#### 3.1.2 房源定位（评论发布时）
- 选择楼栋号
- 选择单元号
- 选择房间号
- 支持模糊定位（仅小区级别）

**设计说明**：
- 用户发布评论时需要精确到房间号
- 评论展示、浏览、统计均以小区为单位进行
- 房间号信息用于数据溯源和防刷评论

### 3.2 评论功能
#### 3.2.1 评论类型
- **避雷评论**：记录负面体验
- **推荐评论**：记录正面体验

#### 3.2.2 评论内容
- 避雷/推荐原因描述
- 标签分类
  - 中介原因
  - 房屋问题
  - 押金问题
  - 环境问题
  - 室友问题
  - 卫生问题
  - 暖气问题
- 图片/视频上传
  - 视频限制：不超过10MB，时长不超过30秒
  - 图片限制：不超过20MB，上传完成后自动压缩
- 匿名选项
- 租期、租金等信息（可选）

#### 3.2.3 评论互动
- 点赞功能
- 回复功能
- 举报功能

#### 3.2.4 评论展示规则
- 评论展示以小区为单位进行
- 用户昵称后显示房源信息：
  - 默认显示模糊信息（如：张三 - XX小区-1号楼）
  - 用户可选择是否公开完整房间号（如：张三 - XX小区-1号楼-2单元-301）
  - 已验证评论显示"已认证"标识，不显示完整房间号

### 3.3 合同验证功能
#### 3.3.1 验证方式
- 上传租房合同照片
- OCR自动识别合同信息
  - OCR识别失败后，用户可以手动输入合同信息
- 人工审核
- 验证通过后标记为"真实评论"
- 允许用户重新提交合同，直到验证通过

#### 3.3.2 验证信息
- 租期验证
- 房东信息验证（脱敏处理）
- 房源信息验证

#### 3.3.3 评论标识
- **已验证评论**：显示"真实评论"标识，纳入数据统计
- **未验证评论**：显示"仅供参考"提醒

### 3.4 数据统计功能
**设计说明**：评论展示、浏览、统计均以小区为单位进行

#### 3.4.1 小区维度统计
- 小区评分
- 避雷率
- 推荐率
- 评论数量
- 平均租金

#### 3.4.2 房源维度统计
- 房间评分（后台数据，用于溯源和防刷）
- 历史评论汇总（后台数据）
- 避雷/推荐趋势（后台数据）

#### 3.4.3 榜单功能
- 热门小区榜
- 避雷榜单
- 推荐榜单
- 推荐机制：根据用户位置和小区评价推荐热门小区和避雷小区
- 个性化推荐：用户提交租房意向（位置、价格范围、房间类型等），系统推荐相关小区
- 实时更新：统计数据实时更新

### 3.5 广告位功能
#### 3.5.1 广告位位置
- 首页Banner：优质小区推流
- 小区详情页
  - 小区周边店铺推荐
  - 小区周边公共服务汇总
  - 小区周边商业服务汇总
- 评论列表页

#### 3.5.2 广告管理
- 广告后台管理系统
- 广告投放统计
- 广告位管理

#### 3.5.3 广告规格
- 广告位尺寸：300x250px
- 展示规则：轮播展示，每个广告位展示30秒
- 计费方式：点击成本和展示成本
  - 广告位展示成本：¥0.05/次展示
  - 广告位点击成本：¥0.10/次点击

## 4. 技术架构

### 4.1 前端架构
#### 4.1.1 移动端
- 技术选型：React Native + TypeScript
- 支持平台：iOS、Android
- 理由：与网页端共享React技术栈，代码复用率高，开发效率高

#### 4.1.2 网页端
- 技术选型：React + TypeScript
- 响应式设计
- 理由：与移动端共享组件逻辑，类型安全，生态成熟

### 4.2 后端架构
#### 4.2.1 服务端
- 技术选型：Node.js + NestJS + TypeScript
- 架构模式：单体架构（MVP阶段，后续可拆分微服务）
- 理由：JavaScript全栈，与前端共享语言，学习成本低

#### 4.2.2 云服务
- 云服务器：阿里云ECS（2核4G × 2台，负载均衡）
- 数据库：PostgreSQL（RDS 2核4G）
- 缓存：Redis（2G）
- 对象存储：阿里云OSS（100GB）
- CDN：阿里云CDN
- 地图服务：高德地图API

### 4.3 性能指标
- 并发用户数：3000人
- 房源容量：5万套单间
- 页面加载时间：< 2秒
- 接口响应时间：< 500ms
- 系统可用性：99.9%

### 4.4 数据库设计
#### 4.4.1 核心数据表
- 用户表（users）
- 小区表（communities）
- 楼栋表（buildings）
- 单元表（units）
- 房间表（rooms）
- 评论表（comments）
- 合同验证表（contracts）
- 广告表（advertisements）
- 点赞表（likes）
- 举报表（reports）
