# 租房避雷App - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 产品定位
租房信息查询平台，帮助租客（大学生、职场人）快速了解周边租房市场的小区和房屋真实情况。

### 1.2 核心价值
- **信息透明**：真实用户评价，避免租房踩坑
- **便捷查询**：快速了解小区详情、老化程度、卫生问题、周边服务
- **防坑指南**：提供中介黑幕、押金问题、房源真实性等避雷信息

### 1.3 目标用户
- 主要用户：正在租房的大学生、职场人
- 管理员：内容审核、系统配置

### 1.4 核心痛点
- 小区安全问题
- 小区设施老化
- 卫生问题
- 中介问题
- 房源真实性
- 押金问题

## 5. 核心功能模块

### 5.1 房源定位功能

#### 5.1.1 小区管理
**功能说明**：管理员维护小区基础信息

**小区信息字段**：
- 基本信息：名称、地址、建成年份、物业公司
- 小区照片：封面图、轮播图
- 服务信息：周边服务、商圈、地铁沿线
- 数据统计：评分、平均租金、绿化率、车位情况
- 分类标签：小区类型、是否有电梯

**小区类型**：
- 普通住宅、公寓、合租、整租、别墅

**首页推荐配置**：
- 管理员可设置首页推荐的小区列表
- 支持排序、上下架操作

**数据来源**：
- 小区基础信息：第三方爬取 + 后台管理
- 地铁/商圈数据：高德地图API

#### 5.1.2 评论定位
**功能说明**：用户发布评论时精确到房源

**定位层级**：
- 小区（必填）
- 楼栋号（必填）
- 单元号（可选）
- 房间号（可选）

**展示规则**：
- 评论展示、浏览、统计均以**小区**为单位进行
- 房间号信息用于数据溯源和防刷评论
- 用户昵称后显示房源信息（如：张三 - XX小区-1号楼）

### 5.2 评论功能

#### 5.2.1 评论类型
- **避雷评论**：记录负面体验
- **推荐评论**：记录正面体验

#### 5.2.2 评论内容
**必填项**：
- 避雷/推荐原因描述
- 标签分类：中介原因、房屋问题、押金问题、环境问题、室友问题、卫生问题、暖气问题

**可选项**：
- 图片上传（不超过20MB，自动压缩）
- 视频上传（不超过10MB，时长不超过30秒）
- 租期、租金信息
- 匿名选项

#### 5.2.3 评论互动
- 点赞功能
- 回复功能
- 举报功能

#### 5.2.4 评论展示
- 按时间倒序展示
- 认证评论显示"真实评论"标识
- 未认证评论显示"仅供参考"
- 支持按标签筛选

### 5.3 搜索筛选功能

#### 5.3.1 搜索方式
- 小区名称搜索
- 地图定位选择
- 热门小区推荐

#### 5.3.2 筛选条件
| 筛选维度 | 选项 | 数据来源 |
|---------|------|---------|
| 价格范围 | 1000-2000、2000-3000、3000-5000、5000+ | 后台配置 |
| 地理位置 | 地铁沿线、商圈 | 高德地图API |
| 小区类型 | 普通住宅、公寓、合租、整租、别墅 | 小区数据 |
| 是否有电梯 | 是/否 | 小区数据 |
| 户型 | 一居室、两居室、三居室、四居及以上 | 小区数据 |

#### 5.3.3 搜索结果
- 小区列表展示（封面图、名称、地址、评分、)
- 支持排序：按评分、按距离、按热度

### 5.4 广告位功能

#### 5.4.1 广告位位置
- 首页Banner
- 小区详情页（侧边栏）
- 评论列表页（插入式）

#### 5.4.2 广告管理
- 广告位管理：添加、编辑、删除广告
- 投放配置：设置投放位置、时间、频次
- 投放统计：展示次数、点击次数、点击率

#### 5.4.3 广告规格
- 尺寸：300x250px
- 展示规则：轮播，每个广告展示30秒
- 计费方式：CPC（0.10元/次点击） + CPM（0.05元/次展示）

## 6. 数据模型

### 6.1 小区模型（Community）
```typescript
{
  id: string
  name: string              // 小区名称
  address: string           // 地址
  coverImage: string        // 封面图
  images: string[]          // 轮播图
  description: string        // 小区描述
  type: string              // 小区类型：普通住宅/公寓/合租/整租/别墅
  hasElevator: boolean      // 是否有电梯
  roomTypes: string[]       // 户型：一居室/两居室/三居室/四居及以上
  buildYear: number         // 建成年份
  propertyCompany: string   // 物业公司
  greenRate: string         // 绿化率
  parking: string           // 车位情况
  rating: number            // 评分（1-5）
  averageRent: number       // 平均租金
  location: {               // 地理位置
    lng: number
    lat: number
  }
  nearbyServices: string[]   // 周边服务
  nearbySubway: string[]     // 地铁沿线
  nearbyBusiness: string[]   // 商圈
  isRecommended: boolean     // 是否首页推荐
  recommendOrder: number     // 推荐排序
  createdAt: Date
  updatedAt: Date
}
```

### 6.2 用户模型（User）
```typescript
{
  id: string
  phone: string
  email: string
  nickname: string
  avatar: string
  wechatOpenId: string
  createdAt: Date
  updatedAt: Date
}
```

### 6.3 评论模型（Comment）
```typescript
{
  id: string
  userId: string
  communityId: string
  building: string          // 楼栋号
  unit: string              // 单元号（可选）
  room: string              // 房间号（可选）
  type: string              // 避雷/推荐
  content: string           // 描述
  tags: string[]            // 标签
  images: string[]          // 图片
  video: string             // 视频
  rentInfo: {               // 租金信息（可选）
    amount: number
    period: string
  }
  rentPeriod: string        // 租期（可选）
  isAnonymous: boolean      // 是否匿名
  isVerified: boolean       // 是否已认证（P2）
  likes: number             // 点赞数
  status: string            // 状态：正常/待审核/已删除
  createdAt: Date
  updatedAt: Date
}
```

### 6.4 广告模型（Advertisement）
```typescript
{
  id: string
  title: string
  image: string
  link: string
  position: string          // 首页Banner/小区详情/评论列表
  status: string            // 状态：上线/下线
  impressions: number        // 展示次数
  clicks: number            // 点击次数
  startDate: Date           // 开始投放时间
  endDate: Date             // 结束投放时间
  createdAt: Date
  updatedAt: Date
}
```

## 7. 技术架构

### 7.1 前端架构
- **框架**：uni-app (Vue 3 + TypeScript)
- **支持平台**：H5、微信小程序
- **理由**：一套代码多端运行，开发效率高；优先开发微信小程序和H5
- **后续扩展**：iOS、Android原生App（P2）

### 7.2 后端架构
- **框架**：NestJS + TypeScript
- **数据库**：PostgreSQL + Prisma ORM
- **缓存**：Redis
- **对象存储**：阿里云OSS
- **地图服务**：高德地图API

### 7.3 部署架构
- **服务器**：阿里云ECS（2核4G × 2台，负载均衡）
- **数据库**：PostgreSQL RDS（2核4G）
- **缓存**：Redis（2G）
- **对象存储**：阿里云OSS（100GB）
- **CDN**：阿里云CDN
