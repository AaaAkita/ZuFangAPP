## 2. 开发环境搭建

### 2.1 前置要求

#### 2.1.1 硬件要求
- **CPU**：4核及以上
- **内存**：8GB及以上
- **硬盘**：100GB及以上
- **网络**：稳定的网络连接

#### 2.1.2 软件要求
- **操作系统**：Windows 10/11、macOS、Linux
- **Node.js**：v18.x 或更高版本
- **npm**：v9.x 或更高版本
- **Git**：v2.x 或更高版本
- **Docker**：v20.x 或更高版本（可选）

### 2.2 安装开发工具

#### 2.2.1 安装Node.js
1. 访问 https://nodejs.org/
2. 下载LTS版本
3. 运行安装程序
4. 验证安装：
```bash
node --version
npm --version
```

#### 2.2.2 安装Git
1. 访问 https://git-scm.com/
2. 下载对应系统版本
3. 运行安装程序
4. 验证安装：
```bash
git --version
```

#### 2.2.3 安装Docker（可选）
1. 访问 https://www.docker.com/
2. 下载Docker Desktop
3. 运行安装程序
4. 验证安装：
```bash
docker --version
docker-compose --version
```

### 2.3 克隆项目代码

```bash
# 克隆项目
git clone https://github.com/your-org/zufang-app.git

# 进入项目目录
cd zufang-app

# 安装依赖
npm install
```

### 2.4 配置环境变量

#### 2.4.1 复制环境变量文件
```bash
# 复制环境变量模板
cp .env.example .env
```

#### 2.4.2 编辑环境变量文件
```bash
# 编辑环境变量
vim .env
```

#### 2.4.3 配置内容
```bash
# 应用配置
NODE_ENV=development
PORT=3000
API_URL=http://localhost:3000

# 数据库配置
DB_HOST=localhost
DB_PORT=5432
DB_NAME=zufang_dev
DB_USER=postgres
DB_PASSWORD=your_password

# Redis配置
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=

# JWT配置
JWT_SECRET=your_jwt_secret
JWT_EXPIRES_IN=7d

# OSS配置
OSS_REGION=oss-cn-hangzhou
OSS_ACCESS_KEY_ID=your_access_key_id
OSS_ACCESS_KEY_SECRET=your_access_key_secret
OSS_BUCKET=zufang-dev

# 第三方服务配置
AMAP_WEB_KEY=your_amap_key
AMAP_ANDROID_KEY=your_amap_android_key
AMAP_IOS_KEY=your_amap_ios_key

# 其他配置...
```

### 2.5 启动开发环境

#### 2.5.1 启动数据库
```bash
# 使用Docker启动PostgreSQL
docker run -d \
  --name postgres-dev \
  -e POSTGRES_DB=zufang_dev \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=your_password \
  -p 5432:5432 \
  postgres:15

# 使用Docker启动Redis
docker run -d \
  --name redis-dev \
  -p 6379:6379 \
  redis:7
```

#### 2.5.2 初始化数据库
```bash
# 运行数据库迁移
npm run db:migrate

# 填充测试数据
npm run db:seed
```

#### 2.5.3 启动后端服务
```bash
# 启动后端服务
npm run dev:backend
```

#### 2.5.4 启动前端服务
```bash
# 启动Web端
npm run dev:web

# 启动移动端
npm run dev:mobile
```

#### 2.5.5 启动管理后台
```bash
# 启动管理后台
npm run dev:admin
```

### 2.6 验证开发环境

#### 2.6.1 访问地址
- **后端API**：http://localhost:3000
- **Web端**：http://localhost:3001
- **移动端**：http://localhost:3002

### 3.5 配置Nginx

#### 3.5.1 创建Nginx配置
```bash
# 创建Nginx配置文件
vim /etc/nginx/sites-available/zufang-test
```

#### 3.5.2 配置内容
```nginx
server {
    listen 80;
    server_name test.zufang.com;

    # 前端静态文件
    location / {
        root /var/www/zufang-app/dist/web;
        try_files $uri $uri/ /index.html;
    }

    # 管理后台
    location /admin {
        root /var/www/zufang-app/dist/admin;
        try_files $uri $uri/ /admin/index.html;
    }

    # 后端API
    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_cache_bypass $http_upgrade;
    }

    # 文件上传
    client_max_body_size 100M;
}
```

#### 3.5.3 启用配置
```bash
# 创建软链接
ln -s /etc/nginx/sites-available/zufang-test /etc/nginx/sites-enabled/

# 测试配置
nginx -t

# 重启Nginx
systemctl restart nginx
```

## 5. CI/CD流程

### 5.1 GitHub Actions配置

#### 5.1.1 创建GitHub Actions工作流
```yaml
# .github/workflows/deploy.yml
name: Deploy

on:
  push:
    branches:
      - main
      - develop

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      - name: Install dependencies
        run: npm ci
      - name: Run tests
        run: npm test
      - name: Run lint
        run: npm run lint

  deploy:
    needs: test
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    steps:
      - uses: actions/checkout@v3
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      - name: Install dependencies
        run: npm ci
      - name: Build
        run: npm run build
      - name: Deploy to server
        uses: easingthemes/ssh-deploy@v4
        with:
          SSH_PRIVATE_KEY: ${{ secrets.SSH_PRIVATE_KEY }}
          REMOTE_HOST: ${{ secrets.REMOTE_HOST }}
          REMOTE_USER: ${{ secrets.REMOTE_USER }}
          TARGET: /var/www/zufang-app
          SOURCE: dist/
```

#### 5.1.2 配置GitHub Secrets
1. 进入GitHub仓库设置
2. 点击"Secrets and variables" -> "Actions"
3. 添加以下Secrets：
   - `SSH_PRIVATE_KEY`：SSH私钥
   - `REMOTE_HOST`：服务器IP地址
   - `REMOTE_USER`：服务器用户名

## 7. 备份和恢复

### 7.1 数据库备份

#### 7.1.1 自动备份
```bash
# 创建备份脚本
vim /var/scripts/backup-db.sh
```

```bash
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/var/backups/postgresql"
DB_NAME="zufang_prod"
DB_USER="zufang_user"
DB_HOST="your-rds-endpoint"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
pg_dump -h $DB_HOST -U $DB_USER $DB_NAME > $BACKUP_DIR/$DB_NAME_$DATE.sql

# 压缩备份文件
gzip $BACKUP_DIR/$DB_NAME_$DATE.sql

# 删除7天前的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete

echo "Backup completed: $BACKUP_DIR/$DB_NAME_$DATE.sql.gz"
```

#### 7.1.2 配置定时任务
```bash
# 编辑crontab
crontab -e

# 添加定时任务（每天凌晨2点备份）
0 2 * * * /var/scripts/backup-db.sh
```

#### 7.1.3 阿里云RDS备份
- **自动备份**：配置自动备份策略
- **备份保留**：保留7天
- **跨地域备份**：开启跨地域备份
